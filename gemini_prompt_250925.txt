flask 프레임워크를 사용해서 웹서비스를 하나 만들거야. 대학교 공과대에서 간단한 물품을 대여하는 대여서비스를 진행하고 있는데, 대여 신청 시 직접 수기로 작성하면서 불편함이 살짝 있어. 그리고 정확한 재고도 학생회 사이에서 공유가 되지않아서 상주 중인 담당 국원 외의 학생회 인원들이 혼선을 빚을 때가 자주 있어. 이어서 얘기하는 포인트들을 참고해서 웹서비스를 하나 개발해줘.

# 전체적인 디자인은 귀엽고 동글동글한 미니멀리즘으로 구성해줘. 색감은 대학교의 대표 색은 짙은 남색이면서 공과대 대표 색깔은 밝은, 파스텔 톤의 하늘색 계열이야. 이 부분을 참고해서 디자인은 알아서 해줬으면 해.

지금 언급한 디렉터리 구조는 그저 예시이며 파일 이름에 얽매이지 않아도 돼. 디렉터리만 보고서는 이해가 되지 않는 부분들은 뒤에서 추가적으로 언급할거야.

# header, footer는 컴포넌트화 해서 유지보수에 용이하도록 제작해줘. 통상 페이지의 헤더, 푸터와 관리자용 헤더, 푸터를 따로 제작할거야. header에는 왼쪽에 간단한 로고가 들어갈거고 오른쪽에는 각 주 메뉴와 커서를 올리면 그에 해당하는 호버 애니메이션이 있고 주 메뉴 텍스트 밑으로 서브메뉴들이 펼쳐지면 돼. 주 메뉴 바의 경우 반응형으로 제작해서 가로비율이 너무 작아지면 햄버거 버튼으로 바꿔서 특히 모바일 환경에서는 햄버거 버튼을 눌러서 오른쪽에서 주 메뉴가 튀어나오고 이때도 마찬가지로 호버링하거나 클릭했을 때 서브메뉴가 펼쳐지면 될 것 같아.

# 통상 페이지와 관리자 페이지의 header의 주 메뉴와 서브메뉴는 네가 임의로 내용을 채워줘. 내가 나중에 수정하도록 할게.

# floating_sns 또한 웹페이지 우하단에 플로팅해서 항상 떠있도록 해줘.

# 각 물품 별 이름과 재고현황은 엑셀파일 stuff_ongoing.xlsx에 저장할거야. 다음은 그 예시야.
물품 | 재고현황
공학용 계산기 | 12
C타입 충전기 | 5
볼펜 | 10

# 메인 페이지에 물품 목록, 즉 이름이 쭉 보이고 물품 별로 오른쪽에 재고 갯수가 보이면 돼. 만약 stuff_ongoing.xlsx에서 재고 갯수가 -1이면 약간 회색빛으로 확인 중 이라고 보이면 될 것 같아.

# 인 당 1개 이상 대여가 불가하지만 여러품목을 한번에 대여하는 건 가능하기 때문에 물품 버튼을 누르면 물픔 별 버튼에 파란색으로 선택됨을 알리는 효과를 주고 테두리도 짙은 파란색으로 변하는 효과를 주자.

# 페이지 밑에는 내가 선택한, 내가 담은 물품들의 목록이 보여주자. 물품 목록이 지난 후 가로줄로 한번 구분감을 주고 "내가 담은 물품"에 소셜커머스 장바구니처럼 내가 선택한 물품들을 이름과 함께 간단하게 보여줘.

# 장바구니 밑에는 대여하기 라는 버튼을 만들고 다음은 확인페이지로 넘어가자.

# 도메인에 다른 인자값이 보이지 않게끔 확인 페이지로 넘어가서 결과적으로 대여 할 물품 목록을 보여주고 밑에는 인증번호와 인적사항을 입력하는 입력란이 있을거야. 인증번호는 0924로 하자. 숫자만을 사용할거고 변수로 처리하여 나중에 바꿀 수 있도록 하자.

# 여기서 인적사항은 이름, 학번, 학과, 전화번호를 입력해야해. 학과도 엑셀파일로 미리 있을거고 드롭다운으로 선택하도록 하자. 해서 입력된 정보들은 borrow_log.xlsx에 저장할거야. 대여물품이 여러개라면 쉼표로 구분하여 입력되면 되고 대여시각은 초단위까지 저장해줘. 엑셀의 양식은 다음과 같아.
이름 | 전화번호 | 학번 | 학과 | 대여물품 | 대여시각

# 마지막으로 확인 페이지에는 다음과 같이 확인 메시지를 보여주자.
yyyy-mm-dd
- 공학용 계산기
- C타입 충전기
대여 확인되었습니다!
""yyyy-mm-dd""까지 반납 부탁드립니다~

# 관리자 페이지는 도메인/setting 으로 연결되면 되고 setting.html에는 로그인을 할건데 그냥 비밀번호만 입력하면 로그인이 되도록 할거야. 일단 비밀번호는 saegil0924로 하자.

# 도메인/setting/main setting_main.html에서는 금일 대여건수, 최근 대여로그 몇가지 정도로 간단하게 엑셀파일들의 정보를 활용해서 보기 좋을 정보들을 간략하게 보여줘.

# header_setting.html은 다 비슷하지만 주 메뉴의 내용이 달라. 재고현황, 대여로그가 보이면 될 것 같아.

# 재고현황의 경우 재고 현황이 보일 뿐만 아니라 여기서 재고 갯수를 변경 할 수 있도록 하자. 여기서 -1을 입력하면 앞선 대여 화면에서는 확인 중으로 뜨는거지. 맨 밑에 + 버튼이 있고 그 버튼을 누르면 새로운 품목을 추가 할 수 있도록 하자. 역시 재고 갯수는 -1이든 0개든 꼭 입력해야 저장할 수 있도록 하자. 내용을 다 입력하면 저장 버튼을 눌렀을 때 재고 갯수가 stuff_ongoing.xlsx에 저장되면 돼. 역시 신규 재고는 엑셀에 밑에 새로 추가하면 돼.

# 대여로그 페이지에서 일단은 단순하게 모든 내용을 엑셀파일의 내용을 불러와서 보여주도록 해줘. 그리고 엑셀의 내용 위에 "대여 실적 엑셀파일 다운로드"라는 버튼을 만들어서 엑셀파일을 바로 받을 수 있도록 하자.
